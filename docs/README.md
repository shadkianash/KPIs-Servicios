# Technical Documentation Root

Welcome to the technical documentation of the **Cyber Services Analytics Platform (CSAP)**. This directory is organized into thematic sections to ensure developers, architects, and operators find precise, well-structured manuals.

---

## Documentation Sections

1. **[Architecture Design (`docs/architecture/`)](architecture/README.md)**: Details structural styles, design decisions, data loaders, and high-level platform diagrams.
2. **[API Reference (`docs/api/`)](api/README.md)**: Documents REST endpoint patterns, query schemas, JSON inputs/outputs, and integration contracts.
3. **[Database Design (`docs/database/`)](database/README.md)**: Maps out the relational schemas, table dictionaries, foreign keys, and indexes.
4. **[System Runbooks (`docs/runbooks/`)](runbooks/README.md)**: Step-by-step procedures to run, test, deploy, and troubleshoot services.

---

## Language & Contribution Rule

All documents inside the `/docs` tree must be authored strictly in **English** using clean, semantic Markdown. If you add components, modify tables, or adjust APIs, make sure to immediately keep the corresponding documentation updated.
